Critical Docker Requirements

Your solution must use Docker Compose with LocalStack running as a container for AWS services, a Vector DB container (Qdrant, Weaviate, or Chroma), a shared Docker network for Lambda-to-service communication, and the LAMBDA_DOCKER_NETWORK environment variable configured correctly. LocalStack must be running in Docker to properly simulate Lambda functions — the pip-installed version will not work for this assessment.

The Docker networking must enable Lambda functions to communicate with both LocalStack services and the vector database. This requires careful configuration of the LAMBDA_DOCKER_NETWORK environment variable to match your compose network name, proper service discovery using container names, and appropriate port mappings for external access.

Required Data Flow Architecture

Your AI instructions must implement this exact five-step flow:

  1. API Gateway → Lambda (Event Handler): Configure POST /events endpoint using the structure from sample-events.json, with request validation schema and proper error responses
  2. Lambda validates & generates embedding: Implement input validation logic, embedding generation using Hugging Face models or embedding service, and comprehensive error handling
  3. Lambda stores data in three locations: DynamoDB for metadata and fast queries, S3 for raw event archival and compliance, Vector DB for semantic similarity search
  4. Lambda publishes to SNS topic: Configure topic with proper permissions and structured message format for downstream consumers
  5. SNS fans out to subscribers: Analytics Lambda for aggregations, Recommendation update Lambda for model improvements, Alert Lambda for anomaly detection

System Design Requirements

The architecture must be designed to scale to 10 million events per day with complete multi-tenant data isolation between companies. Implement event sourcing with S3 as the immutable event store and ensure high availability through fault-tolerant design with fallback strategies.

Multi-tenancy must be enforced at every layer with companyId-based partitioning in DynamoDB, S3 prefix isolation per company, vector database namespaces or metadata filtering, and SNS message attributes for routing.

Infrastructure as Code Specifications

Your Serverless Framework configuration must include Lambda function definitions, DynamoDB tables, S3 buckets, SNS topics, API Gateway endpoints, and IAM roles.

The LocalStack configuration must specify all required AWS services, proper endpoint configuration for local development, Lambda executor settings for Docker-based execution, and volume mounts for data persistence across restarts.

Vector Search Implementation

Implement semantic search capabilities including text embedding generation for all events, k-nearest neighbor search with configurable k value, metadata filtering by company, industry, and stage, hybrid search combining vector similarity with keyword matching, and relevance scoring with configurable thresholds.

The vector database must support collection/index creation with proper dimensions, batch insertion for initial data load, real-time updates as new events arrive, and efficient similarity search with metadata filtering.

AI Executability Test

Your IMPLEMENTATION.md will be tested by pasting it into an AI coding assistant and verifying that the generated code runs successfully. The AI tool should achieve a working system in under 30 minutes with all Docker containers starting successfully, Lambda functions properly communicating with all services, and the complete data flow working end-to-end.

Validation Requirements

When an AI tool receives your IMPLEMENTATION.md, it must be able to start from an empty directory and create a fully functional system. The system must successfully start all Docker containers, deploy Lambda functions to LocalStack, handle API requests correctly, process events through all five stages, and pass automated tests.

The validation will verify that events flow from API Gateway through to SNS subscribers, data is correctly stored in all three storage layers, vector search returns relevant recommendations, multi-tenant isolation is maintained, and error scenarios are handled gracefully.

Common Pitfalls to Avoid

Ensure LocalStack runs in a Docker container rather than pip installation, as Lambda execution requires Docker-based LocalStack. Configure networking properly so Lambda functions can reach both LocalStack services and the vector database. Use container names for service discovery rather than localhost or IP addresses.

Avoid vague instructions that leave room for interpretation. Specify exact package versions, complete configuration files, full error handling logic, and precise deployment commands. Don't assume the AI knows architectural patterns — explicitly define every component, connection, and data flow.
